This page describes how to contribute to Rtd, and provides additional information about participating in the Rtd community.

## Join the community {#join-the-community}

To connect with the Rtd community, join our [Discord](https://discord.gg/rtd).

## Open issues {#open-issues}

To report an issue with Rtd, [create an issue](https://github.com/LinkUVerse/rtd/issues/new/choose) in the GitHub repo. Click **Get started** to open a template for the type of issue to create.

## Fork Rtd to contribute {#fork-rtd-contribute}

To contribute to Rtd source code or documentation, you need only a GitHub account. You can commit updates and then submit a PR directly from the Github website, or create a fork of the repo to your local environment and use your favorite tools to make changes. Always submit PRs to the `main` branch.

See [Rtd Environment Setup](./rtd-environment.mdx) for instructions on forking the Rtd repository, if necessary.

## Contribute via the Rtd Improvement Proposal (SIP) process {#SIP}

The Rtd Network is an open source, decentralized, and permissionless protocol that welcomes community contributions. If you have an idea regarding a core protocol upgrade, the best way to make your voice heard is via submitting a Rtd Improvement Proposal (SIP). For more information on SIPs, see [Contribute to Rtd through SIPs](https://blog.rtd.io/rtd-improvement-proposals-sips/).
