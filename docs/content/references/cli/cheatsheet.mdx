## Addresses & aliases

<table class="w-100">
	<thead>
		<tr>
			<td>Command</td>
			<td>Description</td>
		</tr>
	</thead>
	<tbody>
		<tr>
			<td class="w-2/3">`rtd client active-address`</td>
			<td class="w-1/3">Get the active address</td>
		</tr>
		<tr>
			<td class="w-2/3">`rtd client addresses`</td>
			<td class="w-1/3">List the addresses, their aliases, and the active address</td>
		</tr>
		<tr>
			<td class="w-2/3">`rtd client new-address ed25519`</td>
			<td class="w-1/3">Create a new address with ED25519 scheme</td>
		</tr>
		<tr>
			<td class="w-2/3">`rtd client new-address ed25519 MY_ALIAS`</td>
			<td class="w-1/3">Create a new address with ED25519 scheme and alias</td>
		</tr>
		<tr>
			<td class="w-2/3">`rtd client switch --address ADDRESS`</td>
			<td class="w-1/3">Make this the active address (accepts also an alias)</td>
		</tr>
		<tr> 	
			<td class="w-2/3">`rtd keytool convert PRIVATE_KEY`</td>
			<td class="w-1/3">Convert private key in Hex or Base64 to new format (Bech32 encoded 33 byte flag || private key starting with "rtdprivkey")</td>
		</tr>
		<tr>
			<td class="w-2/3">`rtd keytool generate ed25519`</td>
			<td class="w-1/3">Generate a new keypair with ED25519 scheme and save it to file</td>
		</tr>
		<tr> 	
      <td class="w-2/3">`rtd keytool import INPUT KEY_SCHEME`</td>
			<td class="w-1/3">Add a new key to Rtd CLI Keystore using either the input mnemonic phrase or a Bech32 encoded 33-byte flag || privkey starting with "rtdprivkey"</td>
		</tr>
		<tr>
			<td class="w-2/3">`rtd keytool update-alias OLD_ALIAS NEW_ALIAS`</td>
			<td class="w-1/3">Update the alias of an address</td>
		</tr>
	</tbody>
</table>

			<td class="w-2/3">`rtd client gas ADDRESS`</td>
			<td class="w-1/3">List the gas coins for the given address (accepts also an alias)</td>

</table>

## Create, build, and test a Move project

<table class="w-100">
	<thead>
		<tr>
			<td>Command</td>
			<td>Description</td>
		</tr>
	</thead>
	<tbody>
		<tr>
			<td class="w-2/3">`rtd move build`</td>
			<td class="w-1/3">Build the Move project in the current directory</td>
		</tr>
		<tr>
			<td class="w-2/3">`rtd move build --path PATH`</td>
			<td class="w-1/3">Build the Move project from the given path</td>
		</tr>
		<tr>
			<td class="w-2/3">`rtd move migrate PATH`</td>
			<td class="w-1/3">Migrate to Move 2024 for the package at provided path</td>
		</tr>
		<tr>
			<td class="w-2/3">`rtd move new PROJECT_NAME`</td>
			<td class="w-1/3">Create a new Move project in the given folder</td>
		</tr>
		<tr>
			<td class="w-2/3">`rtd move test`</td>
			<td class="w-1/3">Test the Move project in the current directory</td>
		</tr>
		<tr>
			<td class="w-2/3">`rtd move test --trace`</td>
			<td class="w-1/3">Create an execution trace for the Move tests in the current directory. Use with the [Move Trace Debugger](https://marketplace.visualstudio.com/items?itemName=linku.move-trace-debug) extension.</td>
		</tr>
	</tbody>
</table>

## Executing transactions

<table class="w-100">
	<thead>
		<tr>
			<td>Command</td>
			<td>Description</td>
		</tr>
	</thead>
	<tbody>
		<tr>
			<td class="w-2/3">`rtd client call \`<br/>&nbsp;&nbsp;`--package PACKAGE \`<br/>&nbsp;&nbsp;`--module MODULE \`<br/>&nbsp;&nbsp;`--function FUNCTION`</td>
			<td class="w-1/3">Call a Move package</td>
		</tr>
		<tr>
			<td class="w-2/3">`rtd client merge-coin \`<br/>&nbsp;&nbsp;`--primary-coin COIN_ID \`<br/>&nbsp;&nbsp;`--coin-to-merge COIN_ID`</td>
			<td class="w-1/3">Merge two coins</td>
		</tr>
		<tr>
			<td class="w-2/3">`rtd client split-coin \`<br/>&nbsp;&nbsp;`--coin-id COIN_ID \`<br/>&nbsp;&nbsp;`--amounts 1000`</td>
			<td class="w-1/3">Split a coin into two coins: one with 1000 MIST and the rest</td>
		</tr>
		<tr>
			<td class="w-2/3">
				`rtd client pay-rtd \`<br/>&nbsp;&nbsp;`--input-coins COIN_ID \`<br/>&nbsp;&nbsp;`--recipients ADDRESS \`<br/>&nbsp;&nbsp;`--amounts 100000000`
			</td>
			<td class="w-1/3">Transfer 0.1 RTD to an address and use the same coin for gas</td>
		</tr>
		<tr>
			<td class="w-2/3">`rtd client transfer-rtd \`<br/>&nbsp;&nbsp;`--rtd-coin-object-id COIN_ID \`<br/>&nbsp;&nbsp;`--to ADDRESS`</td>
			<td class="w-1/3">Transfer RTD object to an address and use the same coin for gas</td>
		</tr>
	</tbody>
</table>

## Programmable transaction blocks (PTBs)

<table class="w-100">
	<thead>
		<tr>
			<td>Command</td>
			<td>Description</td>
		</tr>
	</thead>
	<tbody>
		<tr>
			<td class="w-2/3">`rtd client ptb --move-call p::m::f "<type>" args`</td>
			<td class="w-1/3">Call a Move function from a package and module</td>
		</tr>
		<tr>
			<td class="w-2/3">`rtd client ptb --make-move-vec "<u64>" "[1000,2000]"`</td>
			<td class="w-1/3">Make a Move vector with two elements of type u64</td>
		</tr>
		<tr>
			<td class="w-2/3">
				`rtd client ptb \`<br/>&nbsp;&nbsp;`--split-coins gas "[1000]" \`<br/>&nbsp;&nbsp;`--assign new_coins \`<br/>&nbsp;&nbsp;`--transfer-objects
				"[new_coins]" ADDRESS`
			</td>
			<td class="w-1/3">Split a gas coin and transfer it to address</td>
		</tr>
		<tr>
			<td class="w-2/3">`rtd client ptb --transfer-objects "[object_id]" ADDRESS`</td>
			<td class="w-1/3">
				Transfer an object to an address. Note that you can pass multiple objects in the array
			</td>
		</tr>
		<tr>
			<td class="w-2/3">
				`rtd client ptb \`<br/>&nbsp;&nbsp;`--move-call rtd::tx_context::sender \`<br/>&nbsp;&nbsp;`--assign sender \`<br/>&nbsp;&nbsp;`--publish "." \`<br/>&nbsp;&nbsp;`--assign
				upgrade_cap \`<br/>&nbsp;&nbsp;`--transfer-objects "[upgrade_cap]" sender`
			</td>
			<td class="w-1/3">Publish a Move package, and transfer the upgrade capability to sender</td>
		</tr>
		<tr>
			<td class="w-2/3">`rtd client ptb --move-call p::m::f "<type>" args --dry-run`</td>
			<td class="w-1/3">Simulate a PTB execution without committing the transaction</td>
		</tr>
		<tr>
			<td class="w-2/3">`rtd client ptb --move-call p::m::f "<type>" args --preview`</td>
			<td class="w-1/3">Preview PTB commands instead of executing the transaction</td>
		</tr>
		<tr>
			<td class="w-2/3">`rtd client ptb --move-call p::m::f "<type>" args --summary`</td>
			<td class="w-1/3">Show a short summary for a PTB (digest, status, gas cost)</td>
		</tr>
		<tr>
			<td class="w-2/3">
				`rtd client ptb \`<br/>&nbsp;&nbsp;`--move-call p::m::f "<type>" args \`<br/>&nbsp;&nbsp;`--serialize-unsigned-transaction`
			</td>
			<td class="w-1/3">Serialize the unsigned PTB as base64 instead of executing it</td>
		</tr>
	</tbody>
</table>
